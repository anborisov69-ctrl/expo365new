# Buyer Cabinet - Кабинет покупателя EXPO 365

## Обзор

Buyer Cabinet - это персональный кабинет для пользователей-покупателей платформы EXPO 365, предназначенный для управления тендерами, отслеживания поставщиков и получения персональных предложений.

## Основные компоненты

### 1. Dashboard (`/horeca/buyer/dashboard`)

**Z-образная структура дашборда:**
- **Header Section**: Приветствие + кнопка "Создать запрос на покупку"
- **Stats Grid**: 4 метрики в сетке (диагональ Z)
- **Main Content Grid**: 3 колонки (нижняя горизонталь Z)

**Цветовая схема:**
- Primary Color: `#0B2B5E` (Deep Blue) - навигация, заголовки
- Action Color: `#F26522` (Vibrant Orange) - CTA кнопки, акценты

### 2. Tender Creation Module

**Форма "Создать запрос на покупку":**
- Что ищем (обязательное поле)
- Категория (выпадающий список)
- Объём заказа
- Желаемый срок поставки (календарь)
- Тип оплаты (радио кнопки):
  - Предоплата 100%
  - Рассрочка
  - Оплата после поставки
- Дополнительные требования (опционально)

**Логика:**
- После публикации запрос попадает в общую базу Тендерного центра
- Становится видимым для соответствующих экспонентов

### 3. Subscription Feed

**Компонент "Мои поставщики":**
- Переключение между вкладками "Новости" и "Подписки"
- Вывод логотипов поставщиков в сетке 4x4
- Лента новостей с изображениями и описанием
- Индикаторы промо-материалов

### 4. Special Offers Section

**Блок "Ваши персональные условия":**
- Подтягивает данные из `SpecialTerms` по ID покупателя
- Отображение скидок в % и рублях
- Условия применения (минимальная сумма заказа)
- Срок действия предложения с цветовыми индикаторами
- Статистика предложений

## Система безопасности

### Контроль доступа по ролям

**Разрешённые роли:**
- `'buyer'` - основная роль покупателя
- `'partner'` - партнёрский статус с расширенными возможностями

**Механизм защиты:**
```typescript
const { user, isAuthorized, isLoading } = useAuth()

// Редирект неавторизованных пользователей
if (!isAuthorized || !user) {
  redirect('/horeca?action=login')
}

// Проверка ролей
if (user.role !== 'buyer' && user.role !== 'partner') {
  redirect('/horeca?action=unauthorized')
}
```

## Row Level Security (RLS)

**Принципы мультитенантности:**
- Тендеры видны только создателю и релевантным поставщикам
- Персональные условия доступны только конкретному покупателю  
- Новости поставщиков фильтруются по подпискам
